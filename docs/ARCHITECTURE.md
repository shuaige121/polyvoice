# Architecture

## Goals

1. **Streaming first** — TTS plays as it generates; STT returns partial hypotheses where possible.
2. **Pluggable backends** — add a new TTS/STT model in ~100 lines of adapter code.
3. **Chinese-English code-switching** native — Singapore-style `"帮我 deploy 这个 pod"` handled without G2P errors.
4. **Claude Code integration** via OpenAI-compatible API → works with existing ecosystem (voicemode, any client).
5. **Vocabulary-aware STT** — user-specific terms from Claude Code chat history boost recognition.

## Non-goals

- Training / fine-tuning models.
- Wake-word / always-on mic (push-to-talk or explicit activation only).
- Cloud-only setups.

## Components

### 1. Backend adapters (`src/polyvoice/backends/`)

```
tts/
  base.py       # TTSBackend Protocol
  cosyvoice3.py # primary
  spark_tts.py
  fish_speech.py
  edge_tts.py   # cloud fallback
stt/
  base.py       # STTBackend Protocol
  sensevoice.py # primary
  faster_whisper.py
  paraformer.py
```

Each adapter runs in its **own venv** (managed by `uv`) because model deps conflict (CosyVoice wants torch 2.3.1, SenseVoice/sherpa-onnx has different requirements, etc.). Adapters are launched as **subprocess workers** and communicate with the server over Unix domain sockets (JSON lines protocol).

### 2. Servers (`src/polyvoice/server/`)

- `tts.py` — FastAPI, exposes `POST /v1/audio/speech` (OpenAI-compatible) + `GET /voices` + `GET /health`. Streams chunked audio.
- `stt.py` — FastAPI, exposes `POST /v1/audio/transcriptions`. Accepts hotwords via custom header.

Single active backend at a time (VRAM budget); switching reloads the worker.

### 3. Vocabulary system (`src/polyvoice/vocab/`)

```
master.jsonl          # canonical: {phrase, lang, pinyin, weight, category, aliases, count, first_seen}
sources/              # raw extractions
corrections.jsonl     # STT mis-recognitions → correct form
adapters/             # auto-generated per-backend format
  sensevoice.txt
  sherpa_onnx.txt     # with :weight
  capswriter_hot_en.txt
```

Pipeline:
1. `scan.py` walks `~/.claude/projects/**/*.jsonl`, extracts `role=user` and `role=assistant` text.
2. Shards session batches across parallel Sonnet agents (Anthropic SDK).
3. Agents return JSON arrays of `{phrase, lang, category}`.
4. `merge.py` dedupes into `master.jsonl`, updates counts + aliases.
5. `gen_adapters.py` writes per-backend files.
6. STT server reloads adapter file on next request.

### 4. Claude Code integration

**Primary path**: use existing `voicemode` MCP server, point it at our OpenAI-compatible endpoints via `OPENAI_API_BASE=http://127.0.0.1:<port>`.

**Fallback path** (no voicemode): `say-zh` CLI + Claude Code Stop hook + manual dictation via CapsWriter-Offline.

**Voice-mode prompt**: `session_init` hook injects guidance ("be concise, no markdown, use English for technical terms"). Activated by flag file `~/.config/polyvoice/active`.

**What-to-speak decision** (Stop hook):
- `voice_mode` off → skip
- Claude already called `say-zh` this turn → skip (avoid duplicate)
- Last assistant message >400 chars AND >40% code → play "已更新" chime only
- Long but conversational → summarize via Haiku before TTS
- Otherwise → strip markdown → stream to TTS server

## Streaming protocol

### TTS
- Response: `Transfer-Encoding: chunked`, `Content-Type: audio/wav`
- First chunk: streaming WAV header (`data` size = `0xFFFFFFFF`)
- Subsequent chunks: raw PCM16LE frames as generated by the backend
- Client (`ffplay`, `mpv`, or voicemode) plays progressively

### STT
- Input: WAV / webm / raw PCM
- Response: JSON `{text, segments, language}` (OpenAI Whisper format)
- Streaming partial hypotheses: optional future, via SSE on `/v1/audio/transcriptions/stream`

## Config

Single `~/.config/polyvoice/config.toml`:

```toml
[tts]
backend = "cosyvoice3"
port = 7891
default_voice = "f1"

[tts.backends.cosyvoice3]
venv = "venvs/cosyvoice"
model = "/mnt/.../Fun-CosyVoice3-0.5B-2512"
fp16 = true

[stt]
backend = "sensevoice"
port = 7892
hotwords_file = "vocab/adapters/sensevoice.txt"

[stt.backends.sensevoice]
venv = "venvs/sensevoice"
model = "/mnt/.../SenseVoiceSmall-onnx-fp16"

[voice_mode]
flag_file = "~/.config/polyvoice/active"
max_tts_chars = 400
summarize_model = "claude-haiku-4-5"
```
